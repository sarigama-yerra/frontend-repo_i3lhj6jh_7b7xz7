CloudFlow – Installation Guide

This template is a pure HTML + TailwindCSS (CDN) landing page with AOS animations. No build step required.

1. How to edit HTML

Open index.html in your code editor. Update text in each section: Hero, Features, About, Pricing, Testimonials, FAQ, CTA, and Footer. Use semantic tags and preserve Tailwind utility classes for design consistency.

2. How to edit Tailwind

TailwindCSS is loaded via the CDN. You can adjust styles using utility classes directly in HTML. Additional custom styles are stored in assets/css/style.css for reusable patterns (buttons, cards, etc.).

3. How to change images

Replace placeholder image URLs (https://via.placeholder.com/...) in the HTML with your own assets. You can place local images inside assets/img/ and update the src attributes to point at those paths.

4. How to customize colors

Primary theme colors are configured in a small Tailwind setup script inside index.html. Update the color values (primary 500/600/700) to match your brand. You can also extend components in assets/css/style.css.

5. How to deploy on hosting

  • Static hosting: Upload the entire cloudflow-landing folder to Netlify, Vercel, GitHub Pages, or any static host.
  • Ensure all relative paths remain intact (e.g., assets/js/main.js).
  • No server required – everything runs in the browser.

6. JavaScript functionality

The file assets/js/main.js contains: mobile menu toggle, smooth scrolling, FAQ accordion behavior, and AOS initialization. You may extend it for additional interactions.

7. Notes

  • Keep HTML accessible with alt attributes and landmarks.
  • Use descriptive titles and meta descriptions for SEO.
  • Compress images before using in production for optimal performance.
